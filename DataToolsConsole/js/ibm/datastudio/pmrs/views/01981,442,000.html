      Hi Quy,

      First off I would like to talk about the 'New Connection' Wizard. All the information the customer provides in this wizard is converted by us into the 'Connection URL:' which can be seen at the bottom of the wizard. Once the user hits 'Test Connection' or 'Finish' we pass this information to the connection driver and let it do it's work in trying to establish the connection. It is not Data Studio that is creating the connection it is the connection driver and in this case the connection driver is not ours it is 'Microsoft SQL Server 2008 JDBC Driver'.

      To address your comment "Do we support "Integrated Authentication" for SQL Server 2008?"
      I don't think Data Studio cares how the database connection is established as long as it is established. We do support SQL Server 2008 in the sense that after you connect to it we can load objects etc. etc. but the connection itself is up to the Microsoft driver which has to be downloaded seperately from Data Studio.

      I tried a few use case scenarios.
      I connected normally to a SQL Server 2005 database using the Microsoft driver and the connection was successful. I then changed the connection profile and selected the 'Use integrated authentication' checkbox and when I tried connecting I got the following error. This error occurs when the driver is trying to establish the connection. I am not familiar with Microsofts 'Use integrated authentication' feature but I'm sure some other setup is required before this works...


      I connected normally to a SQL Server 2008 database using the Microsoft driver and the connection was successful.       8. ** Dec 10 2012 14:51:51 EST - Chris Khoun/Toronto/IBM@IBMCA - ckhoun@ca.ibm.com **

      Hi Quy,

      My initial investigation on this.

      From what I've read it seems setting this up should be fairly straight forward. Other than having the Microsoft JDBC Driver to use "Integrated Authentication" we also need the sqljdbc_auth.dll file which comes with the driver download.

      Here is the setup that I found.

      http://msdn.microsoft.com/en-us/library/ms378428%28v=sql.100%29.aspx
      Under header Connecting with Integrated Authentication

      The customer previously mentioned "I believe that the problem lies in the location that these files should be placed, especially the sqljdbc_auth.dll file."

      It seems they have to place the sqljdbc_auth.dll file in the same directory as their Microsoft JDBC Driver. (Note the below paths are based off of the screenshot supplied by the customer).

      sqljdbc_auth.dll location.
      C:\Program Files\Microsoft JDBC Driver 4.0 for SQL Server\sqljdbc_4.0\enu\auth\x86\sqljdbc_auth.dll

      sqljdbc4.jar driver location.
      C:\Program Files\Microsoft JDBC Driver 4.0 for SQL Server\sqljdbc_4.0\enu\sqljdbc4.jar

      So copy the ...\enu\auth\x86\sqljdbc_auth.dll file into ...\enu\sqljdbc_auth.dll. There is also an alternative way to specify it through eclipse using -Djava.library.path=C:\Microsoft SQL Server JDBC Driver\sqljdbc_<version>\enu\auth\x86.

      I will give all of this a try now and see if I can get it to work and then I'll reply again.

      Regards,
      Chris

      6. I then tried following the instructions under header Connecting with Integrated Authentication in http://msdn.microsoft.com/en-us/library/ms378428%28v=sql.100%29.aspx

      6.1. I copied the sqljdbc_auth.ddl file from C:\Users\IBM_ADMIN\Microsoft JDBC Driver 4.0 for SQL Server\sqljdbc_4.0\enu\auth\x86\sqljdbc_auth.ddl and pasted it into C:\Users\IBM_ADMIN\Microsoft JDBC Driver 4.0 for SQL Server\sqljdbc_4.0\enu\sqljdbc_auth.ddl which is where the actual Microsoft JDBC Driver .jar is located.

      6.2. I then tried connecting to a Microsoft SQL Server 2008 database using 'Integrated Authentication' but I still get the same error message.
      6.3. I tried the same thing with the sqljdbc_auth.ddl located under x64 but that didn't make a difference.

      7. From here I tried the second approach mentioned in the header Connecting with Integrated Authentication http://msdn.microsoft.com/en-us/library/ms378428%28v=sql.100%29.aspx
      7.1. Close IDA 8.1 if it is open.
      7.2. Navigate to IDA 8.1s installation directory. ex.  C:\Program Files (x86)\IBM\IDA8.1.0
      7.3. Open Notepad as administrator and then open the eclipse.ini file. ex.  C:\Program Files (x86)\IBM\IDA8.1.0\eclipse.ini
      7.4. Add the following java argument to the bottom of the file and save it. (Please modify the path according to where your sqljdbc_auth.ddl is located)
          -"-Djava.library.path=C:\Users\IBM_ADMIN\Microsoft JDBC Driver 4.0 for SQL Server\sqljdbc_4.0\enu\auth\x86"
          -The reason it has to be at the bottom of the file is to ensure that it comes after the -vmargs parameter. Please see the screenshot below to clarify.


      7.5. Save the file and close it.
      7.6. Launch IDA 8.1.
      7.7. Connect to a Microsoft SQL Server 2008 database using 'Integrated Authentication'

      Now unfortunately I still get an error message but it is different than before.


      So the good thing is that at least the sqljdbc_auth.ddl is being picked up this time. It just seems I am having a new/different issue pertaining to where the SQL Server database is located. I think it would be good to ask the customer to try the steps 7. to 7.7. as they might not have the same problem I am facing now.

      Also as to which sqljdbc_auth.ddl to use the x86 or the x64 the explanation for that is as follows. If you are running a 32-bit Java Virtual Machine (JVM), use the sqljdbc_auth.dll file in the x86 folder, even if the operating system is the x64 version. If you are running a 64-bit JVM on a x64 processor, use the sqljdbc_auth.dll file in the x64 folder.
